## ADDED Requirements

### Requirement: 입장 시 랜덤 팔레트 결정
꽃씨 붓 도구를 선택하는 순간 3~5개의 색상을 랜덤으로 선택하여 팔레트를 구성해야 한다. 같은 세션 내에서는 팔레트가 유지되며, 도구를 다시 선택하거나 화면에 새로 진입하면 새 팔레트가 결정된다.

#### Scenario: 도구 선택 시 팔레트 생성
- **WHEN** 사용자가 꽃씨 붓 도구를 탭하면
- **THEN** 3~5개의 랜덤 색상으로 구성된 팔레트가 결정되고 이후 모든 파티클에 사용된다

#### Scenario: 세션 내 팔레트 유지
- **WHEN** 같은 꽃씨 붓 도구로 여러 획을 그리면
- **THEN** 모든 획의 파티클이 동일한 팔레트 색상 범위 안에서 생성된다

### Requirement: 거리 기반 파티클 생성
손가락이 이동할 때 일정 거리 간격(기본값 20px)마다 파티클 오브젝트 하나를 생성해야 한다. 너무 촘촘하거나 듬성듬성하지 않게 자연스러운 밀도를 유지한다.

#### Scenario: 천천히 이동 시 파티클 분포
- **WHEN** 사용자가 꽃씨 붓으로 화면을 천천히 가로지르면
- **THEN** 이동 경로를 따라 20px 간격으로 파티클이 생성된다

#### Scenario: 손가락 정지 시 파티클 미생성
- **WHEN** 꽃씨 붓으로 터치한 채로 손가락을 움직이지 않으면
- **THEN** 추가 파티클이 생성되지 않는다

### Requirement: 피어나는 파티클 애니메이션
각 파티클은 생성 시점에 크기 0에서 최종 크기(finalSize)로 커지는 애니메이션을 재생해야 한다. 애니메이션 지속 시간은 약 300ms이며, 자연스러운 탄성(elastic 또는 ease-out) 커브를 사용한다.

#### Scenario: 파티클 피어나기
- **WHEN** 꽃씨 붓 획 중 파티클이 생성되면
- **THEN** 해당 위치에서 크기 0으로 시작하여 약 300ms 동안 최종 크기로 커지는 애니메이션이 재생된다

#### Scenario: 애니메이션 완료 후 정적 렌더링
- **WHEN** 파티클 피어나기 애니메이션이 완료되면
- **THEN** 파티클이 최종 크기로 캔버스에 고정되어 정적으로 렌더링된다

### Requirement: 파티클의 캔버스 영구 잔류
꽃씨 붓으로 생성된 파티클은 애니메이션 완료 후 캔버스에 영구히 남아 그림의 일부가 되어야 한다. 사라지거나 페이드아웃되지 않는다.

#### Scenario: 파티클 영구 잔류 확인
- **WHEN** 꽃씨 붓으로 그림을 그린 후 다른 도구로 전환하거나 화면을 스크롤해도
- **THEN** 기존 파티클이 그대로 캔버스에 남아있다

#### Scenario: 저장 후 불러오기 파티클 유지
- **WHEN** 꽃씨 붓이 포함된 그림을 저장하고 다시 불러오면
- **THEN** 모든 파티클의 위치, 모양, 색상, 크기가 동일하게 복원된다

### Requirement: 개발용 기본 파티클 도형 제공
개발 단계에서는 별(star), 하트(heart), 원(circle) 세 가지 도형을 Flutter Canvas API로 직접 그린다. 도형은 랜덤으로 선택되며, 향후 SVG 에셋으로 교체 가능하도록 도형 렌더링 로직을 분리한다.

#### Scenario: 랜덤 도형 선택
- **WHEN** 파티클이 생성될 때
- **THEN** star, heart, circle 중 하나가 랜덤으로 선택되어 렌더링된다

#### Scenario: 도형별 랜덤 회전 적용
- **WHEN** 파티클이 렌더링될 때
- **THEN** 각 파티클에 랜덤한 회전각이 적용되어 단조롭지 않은 배치가 된다
