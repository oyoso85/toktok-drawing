## ADDED Requirements

### Requirement: 색칠 도안 표시
숫자/ABC 색칠 모드 진입 시 내장된 색칠 도안 목록을 표시하고, 선택 시 검정 윤곽선 도안을 캔버스에 렌더링해야 한다(SHALL).

#### Scenario: 도안 목록 표시
- **WHEN** 사용자가 숫자/ABC 색칠 모드에 진입한다
- **THEN** 앱에 내장된 색칠 도안 목록이 썸네일로 표시된다

#### Scenario: 도안 선택 후 캔버스 표시
- **WHEN** 사용자가 도안을 선택한다
- **THEN** 검정 윤곽선 도안이 캔버스에 표시되며, 각 영역에 기호(숫자, 알파벳, 또는 모양 기호)가 표시된다

### Requirement: 기호-색상 매핑 표시
각 도안에는 기호와 색상의 매핑 정보가 하단 팔레트에 표시되어야 한다(SHALL). 기호 유형은 도안별로 숫자(1, 2, 3...), 알파벳(A, B, C...), 또는 모양 기호(☆, ♡, ♤, ◆ 등) 중 하나를 사용할 수 있다.

#### Scenario: 색상 팔레트에 기호 매핑 표시
- **WHEN** 도안이 캔버스에 표시된다
- **THEN** 하단에 기호별 색상 팔레트가 표시된다 (예: 1=빨강, A=노랑, ☆=파랑)

### Requirement: 영역 탭하여 색칠
사용자가 도안의 영역을 탭하면 해당 영역이 기호에 매핑된 색상으로 채워져야 한다(SHALL).

#### Scenario: 올바른 영역 탭
- **WHEN** 사용자가 숫자 "1"이 표시된 영역을 탭한다
- **THEN** 해당 영역이 숫자 "1"에 매핑된 색상으로 채워진다

#### Scenario: 이미 채워진 영역 탭
- **WHEN** 사용자가 이미 색칠된 영역을 다시 탭한다
- **THEN** 색상이 유지되고 변화 없음 (이미 올바르게 채워진 상태)

### Requirement: 색칠 완성 피드백
모든 영역이 색칠되면 완성 피드백을 제공해야 한다(SHALL).

#### Scenario: 모든 영역 색칠 완료
- **WHEN** 도안의 모든 영역이 색칠된다
- **THEN** 완성 축하 애니메이션 또는 효과가 표시된다

### Requirement: 색칠 초기화
사용자는 색칠한 내용을 초기화할 수 있어야 한다(SHALL).

#### Scenario: 전체 초기화
- **WHEN** 사용자가 초기화 버튼을 탭한다
- **THEN** 모든 영역의 색칠이 제거되고 기호가 다시 표시된 초기 상태로 돌아간다
