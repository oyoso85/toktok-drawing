## ADDED Requirements

### Requirement: 모드 선택 화면 표시
앱 실행 시 그리기 모드를 선택하는 화면을 표시해야 한다(SHALL). 각 모드는 큰 아이콘과 짧은 라벨로 구성되며, 3~7세 아이가 직관적으로 선택할 수 있어야 한다. 모드 목록은 확장 가능하게 설계하여 향후 새로운 모드를 추가할 수 있어야 한다(SHALL).

#### Scenario: 앱 최초 실행 시 모드 선택 화면 표시
- **WHEN** 사용자가 앱을 실행한다
- **THEN** 등록된 모드들(초기: 선 따라 그리기, 숫자/ABC 색칠, 대칭 그리기, 자유 그리기)이 카드 형태로 표시된다

#### Scenario: 모드 확장 지원
- **WHEN** 새로운 그리기 모드가 추가된다
- **THEN** 모드 선택 화면에 자동으로 새 모드 카드가 추가되며, 스크롤로 탐색 가능하다

#### Scenario: 모드 카드 구성
- **WHEN** 모드 선택 화면이 표시된다
- **THEN** 각 모드 카드에는 모드를 나타내는 아이콘, 모드 이름, 간단한 미리보기 이미지가 포함된다

### Requirement: 모드 선택 및 진입
사용자가 모드 카드를 탭하면 해당 모드의 드로잉 화면으로 즉시 전환되어야 한다(SHALL).

#### Scenario: 모드 카드 탭하여 진입
- **WHEN** 사용자가 특정 모드 카드를 탭한다
- **THEN** 해당 모드의 드로잉 화면으로 전환된다

#### Scenario: 드로잉 중 모드 선택 화면 복귀
- **WHEN** 사용자가 드로잉 화면에서 홈/뒤로가기 버튼을 탭한다
- **THEN** 현재 드로잉이 자동 저장된 후 모드 선택 화면으로 돌아간다

### Requirement: 유아 친화적 UI
모드 선택 화면은 3~7세 대상에 적합한 UI를 제공해야 한다(SHALL). 텍스트는 최소화하고 아이콘/이미지 중심으로 구성한다.

#### Scenario: 큰 터치 영역
- **WHEN** 모드 선택 화면이 표시된다
- **THEN** 각 모드 카드의 터치 영역은 최소 80x80dp 이상이어야 한다

#### Scenario: 밝은 색상 테마
- **WHEN** 모드 선택 화면이 표시된다
- **THEN** 배경과 카드에 밝고 다채로운 색상이 사용된다

### Requirement: 아이들 모션 효과
모드 선택 화면의 UI 요소(모드 카드, 아이콘, 텍스트, 이미지)는 사용자가 아무 동작을 하지 않아도 살짝 흔들리거나 튀는 등의 미세한 반복 애니메이션을 제공해야 한다(SHALL). 이를 통해 생동감 있고 아이들의 흥미를 끄는 화면을 만든다.

#### Scenario: 모드 카드 유휴 애니메이션
- **WHEN** 모드 선택 화면이 표시되고 사용자가 아무 동작을 하지 않는다
- **THEN** 각 모드 카드가 살짝 흔들리거나 위아래로 미세하게 움직이는 반복 애니메이션이 재생된다

#### Scenario: 아이콘/이미지 유휴 애니메이션
- **WHEN** 모드 선택 화면이 표시된다
- **THEN** 카드 내 아이콘과 미리보기 이미지가 부드럽게 흔들리는 모션을 가진다
